# Evals 框架说明

本文档定义回归评测体系的目标与基本结构。

## 目标

- 验证输出结构是否稳定
- 验证评分逻辑是否出现漂移
- 验证 Prompt 或 Provider 变更是否导致退化
- 为版本比较提供可复用报告

## 评测对象

- 输入结构
- 输出 JSON 合法性
- 顶层评分分布
- 平台推荐结构
- 详细分析字段完整性
- Prompt 版本切换后的稳定性
- Provider 切换后的稳定性

## 评测类型

### 1. 结构合法性评测

检查 JSON 是否符合 schema。

### 2. 评分稳定性评测

检查同类文本在不同版本下的评分漂移。

### 3. 鲁棒性评测

检查异常输入、缺失字段、边缘样本时的系统表现。

### 4. 回归评测

比较当前版本与历史基线之间的变化。

## 评测资产

- `datasets/`：样本与金标准
- `cases/`：评测用例组织
- `runners/`：执行入口
- `reports/`：报告输出
- `baselines/`：历史基线结果

## 原则

- Prompt 变更要跑回归
- Provider 变更要跑回归
- Schema 变更要跑结构合法性验证
