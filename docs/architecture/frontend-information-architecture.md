# 前端信息架构

## 文档定位

本文定义前端页面结构、核心对象、导航语义、页面可达性与预留页面位置，是当前前端信息架构的主文档。

本文收敛原“页面结构规划”“路由与导航规范”“结果对比页预留说明”的信息架构层内容，但不替代页面规格、状态流或字段契约。

## 架构基线

当前前端采用以下结构基线：

- 采用 `工作台主导式` 页面组织方式
- 首页采用 `均衡型轻工作台首页`
- 历史记录页采用 `按任务组织`
- `任务详情 / 状态页` 与 `结果详情页` `分页分离`
- `结果对比页` 仅保留为 `结构预留`

## 采用该基线的原因

- 最符合“评测任务工作台”的产品定位
- 与 `EvaluationTask` 和 `EvaluationResult` 的对象边界一致
- 能稳定承接异步任务、历史沉淀、版本追踪与未来结果对比
- 可以通过“首期核心页 + 结构预留页”的方式控制当前范围

## 核心信息对象

### 1. 输入草稿 `InputDraft`

前端编辑中的输入对象，用于承接用户尚未提交的文本、文件与基础业务参数。

### 2. 稿件对象 `Manuscript`

可提交给后端创建评测任务的输入对象。

### 3. 评测任务 `EvaluationTask`

前端任务流的主对象，用于承载任务标识、输入摘要、执行状态、错误信息与基础元信息。

### 4. 评测结果 `EvaluationResult`

结果页的主对象，用于承载后端确认可展示的正式结构化结果。

### 5. 历史记录项 `HistoryTaskItem`

历史记录页的主列表对象，按任务组织，用于帮助用户定位历史任务并进入任务页或结果页。

### 6. 对比选择对象 `CompareSelection`

为未来 `结果对比页` 预留的选择对象。当前只保留概念位置，不进入首期实现。

## 页面树

建议采用如下页面结构：

```text
工作台首页
├─ 新建评测任务页
├─ 任务详情 / 状态页
│  └─ 结果详情页
├─ 历史记录页
└─ 结果对比页（结构预留）
```

## 概念路由语义

本文给出概念性路由语义，用于帮助后续信息架构与页面规格统一，不视为最终实现约束。

```text
/
/tasks/new
/tasks/{taskId}
/tasks/{taskId}/result
/history
/compare
```

路由含义：

- `/`：`工作台首页`
- `/tasks/new`：`新建评测任务页`
- `/tasks/{taskId}`：`任务详情 / 状态页`
- `/tasks/{taskId}/result`：`结果详情页`
- `/history`：`历史记录页`
- `/compare`：`结果对比页`（结构预留）

## 导航模型

### 一级导航

首期一级导航固定为：

- `工作台首页`
- `新建评测任务`
- `历史记录`

### 二级入口

- 首页快速新建入口 -> `新建评测任务页`
- 首页最近任务摘要 -> `任务详情 / 状态页`
- 首页最近结果摘要 -> `结果详情页`
- 历史记录页任务项 -> `任务详情 / 状态页`
- 历史记录页结果入口 -> `结果详情页`
- `任务详情 / 状态页` 中的结果入口 -> `结果详情页`

### 预留入口

- `结果对比页` 当前不进入一级导航
- 后续可从 `历史记录页` 或 `结果详情页` 提供进入入口

## 页面可达性规则

### 工作台首页

- 默认可达
- 作为统一入口页与摘要页

### 新建评测任务页

- 默认可达
- 作为正式任务流起点

### 任务详情 / 状态页

- 只有在存在合法任务标识时可达
- 无效任务标识不得进入正常详情阅读路径

### 结果详情页

- 只有在任务完成且结果可展示时进入正式结果阅读路径
- 结果不可用、读取失败或被阻断时，不进入正常结果正文路径

### 历史记录页

- 默认可达
- 按任务组织历史入口，不按结果组织首期主列表

### 结果对比页

- 当前仅结构预留
- 首期不进入一级导航
- 不进入当前实现主线

## 页面与对象映射

| 页面 | 主对象 | 页面职责 |
| --- | --- | --- |
| `工作台首页` | `DashboardTaskSummaryView` / `DashboardResultSummaryView` | 提供任务入口、任务摘要、结果快捷入口 |
| `新建评测任务页` | `InputDraft` / `Manuscript` | 采集输入并创建任务 |
| `任务详情 / 状态页` | `EvaluationTask` | 展示任务状态、输入摘要、错误信息与结果入口 |
| `结果详情页` | `EvaluationResult` | 展示正式结构化结果与基础元信息 |
| `历史记录页` | `HistoryTaskItem` | 按任务组织历史记录，支持检索与回访 |
| `结果对比页` | `CompareSelection` | 未来承接结果对比，仅保留位置 |

## 任务完成后的导航规则

当前已确认：

- 任务完成后默认留在 `任务详情 / 状态页`
- 页面提供结果入口
- 由用户手动进入 `结果详情页`
- 不把自动跳转结果页作为当前默认语义

## 结果对比页的预留边界

### 当前定位

`结果对比页` 当前只作为结构预留页面，用于为未来版本差异、任务差异、结果差异提供独立视图。

### 当前不进入首期的原因

- 当前项目仍处于结构建设阶段
- 首期更应优先完成任务流、结果消费与历史沉淀
- 对比页依赖更稳定的结果元信息与历史策略
- 过早推进会扩大首期范围，削弱主流程聚焦

### 未来进入规划前的前置条件

- 任务流与状态流稳定
- 结果展示规则稳定
- 历史记录与检索策略稳定
- 结果元信息扩展策略明确

## 当前不纳入信息架构的内容

- 研究视图页
- Prompt 调试入口
- 模型调试入口
- 复杂运营看板
- 批量任务页
- 多角色后台管理页
