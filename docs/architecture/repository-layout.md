# 仓库结构说明

本文档说明仓库的正式目录边界，目的是为后续开发提供统一的结构约束。

## 顶层结构

- `apps/`：可运行应用入口
- `packages/`：可复用核心能力
- `prompts/`：正式 Prompt 资产
- `evals/`：评测与回归体系
- `docs/`：规划、架构、契约、运维与研究文档
- `output/`：研究与临时产物
- `scripts/`：仓库维护脚本

## 结构原则

### 1. 应用入口与核心能力分离

- `apps/` 负责运行时入口和外部交互
- `packages/` 负责可复用核心能力
- 正式业务规则应优先沉淀到 `packages/` 而不是 `apps/`

### 2. Prompt 与源码隔离但受控

- Prompt 独立存放于 `prompts/`
- Prompt 由后端治理，不由前端消费
- Prompt 变更需要可追踪、可回归

### 3. 研究产物与正式实现隔离

- `output/playwright/` 仅用于研究、抓取和分析
- 该目录不作为正式实现的依赖来源

### 4. 契约优先

- 输入输出结构应以 `packages/schemas/` 和 `docs/contracts/` 为基准
- 所有应用层都必须遵守同一套 JSON 契约

## 当前状态

当前仓库处于结构建设阶段，已完成目录骨架，尚未进入实际业务开发。
