# 评分流水线

本文档定义小说智能打分系统的标准评分流程。

## 标准流程

1. 接收用户输入
2. 校验输入结构与基本边界
3. 构建评分任务对象
4. 选择 Prompt 版本与评分模板
5. 选择 Provider 与执行策略
6. 调用模型生成评分结果
7. 校验严格 JSON 输出
8. 执行结果修正或失败处理
9. 返回正式结构化结果
10. 记录日志与可选评测信息

## 每一步的职责

### 输入接收

- 只处理系统边界输入
- 不在这里处理深层业务规则

### 输入校验

- 校验文本是否存在
- 校验输入类型是否合法
- 校验用户配置结构是否合法

### 任务构建

- 为后续执行统一上下文
- 记录 Prompt、Schema、Provider 的版本信息

### Prompt 选择

- 根据评分任务选择合适 Prompt
- 不允许前端传入正式评分 Prompt 正文

### Provider 执行

- 使用统一抽象接口发起模型调用
- 不将具体供应商细节暴露给上层

### JSON 校验

- 将输出视为必须满足的结构契约
- 非法输出进入失败路径

### 失败处理

- 可以记录错误
- 可以触发重试策略
- 但不得返回无法解析的伪结构化结果

## 设计目标

- 流水线标准化
- 错误路径清晰
- 契约优先
- 支持未来多 Provider 与回归评测
