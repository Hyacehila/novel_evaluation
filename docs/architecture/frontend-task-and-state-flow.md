# 前端任务流与状态流

## 文档定位

本文定义前端围绕评测任务工作台的主流程、状态命名、状态转移、失败与阻断规则，以及页面级空态、错误态与恢复入口约束。

本文是前端状态语义的主锚点。页面规格、术语与联调口径中的状态命名均以本文为准。

## 核心流程原则

- 前端采用 `Workflow-First`
- 主流程以任务为核心，不以结果页为唯一中心
- `任务详情 / 状态页` 与 `结果详情页` 分离
- 任务完成后默认由用户手动进入结果页
- 异常结果不能进入正式结果展示路径
- 摘要页与详情页职责分离

## 核心对象流转

```text
InputDraft -> Manuscript -> EvaluationTask -> EvaluationResult -> HistoryTaskItem
```

说明：

- `InputDraft`：前端输入中的草稿态
- `Manuscript`：可提交的输入对象
- `EvaluationTask`：已提交、可追踪的任务对象
- `EvaluationResult`：后端校验通过的正式结果对象
- `HistoryTaskItem`：历史记录页消费的任务摘要对象

## 主流程

### 流程一：新建任务

1. 用户进入 `新建评测任务页`
2. 用户填写输入内容
3. 前端完成边界内校验
4. 前端提交任务创建请求
5. 后端返回任务标识
6. 前端进入 `任务详情 / 状态页`

### 流程二：任务执行中

1. 前端读取任务状态
2. 任务处于 `queued` 或 `processing`
3. 用户停留在 `任务详情 / 状态页`
4. 页面展示状态信息、输入摘要与基础元信息

### 流程三：任务完成

1. 后端将任务状态更新为 `completed`
2. 前端在 `任务详情 / 状态页` 感知完成
3. 页面提供结果入口
4. 用户手动进入 `结果详情页`
5. 前端读取正式结构化结果

### 流程四：历史回访

1. 用户进入 `历史记录页`
2. 以任务为主对象查看历史项
3. 用户进入对应 `任务详情 / 状态页` 或 `结果详情页`

## 状态体系

### 输入与提交状态

| 状态 | 含义 |
| --- | --- |
| `editing` | 用户正在输入 |
| `validation_failed` | 输入未通过前端边界校验 |
| `submitting` | 任务正在提交 |
| `submit_failed` | 任务提交失败 |

### 任务状态

| 状态 | 含义 |
| --- | --- |
| `queued` | 任务已创建，等待执行 |
| `processing` | 任务执行中 |
| `completed` | 任务执行完成 |
| `failed` | 任务执行失败 |

### 结果状态

| 状态 | 含义 |
| --- | --- |
| `available` | 存在可展示的正式结果 |
| `not_available` | 结果暂不可展示 |
| `fetch_failed` | 结果读取失败 |
| `blocked` | 结果被严格阻断，不能作为正式结果展示 |

### 页面展示态

以下状态用于页面展示，不是领域主对象状态：

- `loaded`
- `empty`
- `filtered_empty`
- `load_failed`

它们主要用于首页摘要区、历史记录页列表与检索反馈。

## 状态转移规则

### 输入阶段

```text
editing -> validation_failed
editing -> submitting
submitting -> submit_failed
submitting -> queued
```

### 任务阶段

```text
queued -> processing
queued -> failed
processing -> completed
processing -> failed
```

### 结果阶段

```text
completed -> available
completed -> not_available
completed -> blocked
available -> fetch_failed
```

## 失败与阻断分类

| 类型 | 含义 | 主要停留页面 | 是否允许进入结果正文 |
| --- | --- | --- | --- |
| 输入校验失败 | 输入内容不满足前端边界 | `新建评测任务页` | 否 |
| 提交失败 | 任务创建请求失败 | `新建评测任务页` | 否 |
| 任务失败 | 任务执行未完成或失败 | `任务详情 / 状态页` | 否 |
| 结果不可用 | 任务完成但结果暂不可读 | `任务详情 / 状态页` 或结果错误态 | 否 |
| 结果读取失败 | 读取正式结果时失败 | `结果详情页` 错误态 | 否 |
| 结果阻断 | 结果不满足正式结果展示条件 | `结果详情页` 阻断态或任务页说明 | 否 |

## 页面行为约束

### 新建评测任务页

- 只承载 `editing`、`validation_failed`、`submitting`、`submit_failed`
- 不展示任务执行状态
- 不展示正式结果

### 任务详情 / 状态页

- 主要承载 `queued`、`processing`、`completed`、`failed`
- 仅在任务完成且结果可展示时提供进入结果页入口
- 当结果处于 `not_available` 或 `blocked` 时，不以伪结果替代任务页说明

### 结果详情页

- 只有 `available` 才进入正式结果阅读态
- `not_available`、`fetch_failed`、`blocked` 不进入正常结果正文
- 非法结果不得以“降级结果”“部分结果”的形式展示

### 工作台首页与历史记录页

- 只承载摘要展示态，不承载正式结果正文
- 空列表或无摘要信息可作为正常空态处理
- 摘要不得替代任务页或结果页的正式语义

## 空态、错误态与恢复入口

### 正常空态

以下应视为正常空态：

- `工作台首页` 暂无最近任务
- `工作台首页` 暂无最近结果摘要
- `历史记录页` 暂无历史任务
- 搜索或筛选后暂无匹配结果

### 非正常空态

以下不应视为普通空态：

- 任务存在但状态读取失败
- 结果存在但读取失败
- 结果被阻断

### 错误态规则

- 输入错误：停留在输入页，指向用户可修正的问题
- 提交错误：停留在输入页，允许重新提交
- 任务错误：停留在任务页，不生成结果入口
- 结果错误：停留在结果错误态，不展示伪结果
- 阻断错误：只展示阻断说明、状态说明与返回入口

### 恢复入口规则

| 页面 | 可恢复动作 |
| --- | --- |
| `新建评测任务页` | 修改输入、重新提交、返回首页 |
| `任务详情 / 状态页` | 等待状态变化、返回首页、返回历史记录页 |
| `结果详情页` | 返回任务页、返回历史记录页 |
| `历史记录页` | 调整搜索筛选条件、返回首页 |

## 阻断态特别规则

当结果处于 `blocked`：

- 不展示正式结果正文
- 不使用“部分可读结果”替代
- 不把阻断包装成正常结果页
- 只允许展示阻断说明、状态说明与返回入口

## 手动进入结果页规则

本项为已确认策略：

- 任务完成后默认留在 `任务详情 / 状态页`
- 由页面提供结果入口
- 由用户手动进入 `结果详情页`
- 不将自动跳转结果页作为当前默认语义
