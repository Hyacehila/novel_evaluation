# 前后端边界说明（前端相关）

## 文档目的

本文用于定义前端与后端在用户交互、任务创建、结果消费、错误处理与结构化输出方面的职责边界，供后续接口设计、页面开发与联调使用。

本文不替代正式 Schema 定义。正式结构定义仍应以 `packages/schemas/` 为唯一真源。

## 核心原则

### 1. Prompt 仅后端治理

前端不得持有、拼接或提交正式评分 Prompt 正文。

前端只能提交联合投稿包输入与必要参数，Prompt 的选择、加载、版本控制与渲染均由后端承担。

首期允许前端提交的基础业务参数仅限非 Prompt 型业务参数，不包括 Prompt 正文、Prompt 版本选择或 Provider 内部策略。

### 2. 契约优先

前端消费的正式结果必须是后端校验通过后的结构化结果。

前端不直接消费模型原始输出，也不直接解析未经后端确认的半结构化文本。

### 3. 任务对象与结果对象分离

前端应将“评测任务”和“评测结果”视为两个相关但不同的对象：

- `EvaluationTask` 由后端负责创建、更新和提供状态
- `EvaluationResult` 由后端在任务完成后产出并提供读取接口

### 4. 严格阻断异常结果

当后端未返回合法结构化结果时，前端采用“严格阻断”策略：

- 不展示伪结果
- 不尝试自行拼装正式结果
- 不对不合法结果做强兜底修复
- 仅展示错误态、任务状态与必要的失败说明

### 5. 基础元信息展示

首期前端页面中，只要求显式展示基础元信息：

- 任务 ID
- 创建时间
- 结果时间

Prompt 版本、Schema 版本、Provider 标识等信息可由后端保留并在后续阶段扩展到前端展示。

## 前端负责什么

### 输入相关

- 采集标题
- 采集章节正文输入
- 采集大纲输入
- 采集文件上传输入
- 采集 `sourceType` 等基础参数
- 在前端边界内做基础校验
- 发起新建任务请求

### 任务相关

- 展示任务状态
- 展示任务基础元信息
- 展示任务失败信息
- 提供进入结果页的入口

### 结果相关

- 展示后端确认可读的结构化结果
- 渲染顶层评分、平台推荐、结论、分析、可视化内容
- 展示基础元信息

### 历史相关

- 展示按任务组织的历史记录
- 提供筛选与回访入口

## 前端不负责什么

- 不持有正式 Prompt
- 不决定 Prompt 版本
- 不适配模型供应商细节
- 不直接发起模型调用
- 不决定结果结构是否合法
- 不修复非法模型输出为正式结果
- 不将研究产物当作正式结果来源

## 后端负责什么

### 任务创建与执行

- 接收联合投稿包输入
- 校验输入结构
- 构建任务对象
- 选择 Prompt 版本
- 选择 Provider 与执行策略
- 维护任务状态

### 结果生成与校验

- 生成结果
- 校验严格 JSON 输出
- 在必要时进入失败路径
- 决定结果是否可作为正式结构化结果返回

### 错误与失败处理

- 返回任务失败状态
- 返回结果不可用状态
- 返回结果阻断状态
- 提供必要错误信息
- 保证前端不会收到“看起来像结果但结构不合法”的正式响应

## 输入边界

### 前端可提交内容

- `title`
- `chapters`
- `outline`
- `sourceType`

### 前端不可提交内容

- 正式 Prompt 正文
- 后端内部执行策略
- 模型内部规则定义
- 旧 `inputType` 主输入枚举

## 结果边界

### 前端可消费内容

- 四个顶层评分
- 平台推荐
- 编辑结论
- 市场判断
- 详细分析
- 优势与弱点
- 可视化所需结构化数据
- 基础元信息

### 前端不可消费为正式结果的内容

- 原始模型流式文本
- 未经校验的半结构化文本
- 缺少关键字段但未明确标注失败的伪结果

## 错误处理边界

### 前端处理方式

前端只处理展示级错误，不承担业务级结果修复责任。

前端应处理：

- 输入校验错误
- 提交失败
- 任务状态查询失败
- 结果读取失败
- 结果不可用
- 结果阻断
- `fetch_failed` 本地派生状态

前端不应处理：

- 非法 JSON 修复
- 字段结构猜测补齐
- 非正式结果推断

### 后端处理方式

后端应处理：

- 输入结构校验
- Prompt 选择与版本治理
- 模型执行
- JSON 校验
- 失败路径判定

## 页面与边界映射

### 工作台首页

- 前端读取摘要信息
- 后端提供可用于首页聚合的任务和结果摘要

### 新建评测任务页

- 前端提交联合投稿包输入
- 后端创建任务并返回任务标识

### 任务详情 / 状态页

- 前端读取任务状态
- 后端提供任务对象与状态信息

### 结果详情页

- 前端读取正式结构化结果
- 后端保证结果已经通过校验并可展示

### 历史记录页

- 前端读取按任务组织的历史项
- 后端提供任务列表与最简筛选能力

## 首期最简边界约定

### 元信息展示

首期仅要求前端显式展示：

- 任务 ID
- 创建时间
- 结果时间

### 历史筛选能力

首期仅要求：

- 搜索
- 任务状态筛选

### 结果异常策略

首期统一采用：

- 严格阻断
- 不展示伪结果
- 用错误态替代不合法结果展示
