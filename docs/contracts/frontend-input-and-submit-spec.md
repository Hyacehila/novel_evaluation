# 前端输入与提交约束

## 文档目的

本文档冻结输入页、上传区和任务提交时前端应遵守的边界。

## 正式输入对象

前端提交的业务输入统一映射到：

- `title`
- `chapters`
- `outline`
- `sourceType`

## 提交模式

前端必须支持两种提交模式：

- `application/json`
- `multipart/form-data`

## 上传字段

`multipart/form-data` 固定字段为：

- `chaptersFile`
- `outlineFile`

## 上传格式与大小

前端只允许：

- `TXT`
- `MD`
- `DOCX`

前端默认按单文件 `10 MiB` 校验，具体上限以后端配置为准。

## 前端负责

- 校验标题是否为空
- 校验正文与大纲至少存在一侧
- 校验文件存在性
- 校验文件格式
- 校验文件大小
- 提交中禁用按钮与防重复点击

## 前端不负责

- 解析文件正文
- 自动拆章节
- 推断正式结果
- 提交 Prompt/Provider 内部参数

## 提交后行为

- 成功后跳转任务页
- 失败后停留输入页
- `blocked` 与 `failed` 由任务页和结果页消费，不在输入页自行推断
