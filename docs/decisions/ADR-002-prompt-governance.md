# ADR-002：Prompt 仅由后端治理

## 状态

已接受。

## 背景

Prompt 是系统评分哲学、结构输出和行为稳定性的核心资产。如果 Prompt 散落在前端、脚本或临时文件中，将难以治理、回滚和评测。

## 决策

- Prompt 正式资产统一存放在 `prompts/`
- Prompt 只允许在后端或后端控制范围内加载与使用
- 前端不得直接持有正式评分 Prompt
- Prompt 必须有版本、元数据和变更记录
- Prompt 变更必须与 Schema 和 Evals 联动考虑

## 结果

- 提高安全性与治理能力
- 提高输出一致性与可追踪性
- 为后续回归评测提供稳定基准
