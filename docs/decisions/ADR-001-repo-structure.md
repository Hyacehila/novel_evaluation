# ADR-001：采用技术栈中立的仓库结构

## 状态

已接受。

## 背景

当前仓库尚处于结构建设阶段，正式业务代码尚未形成。项目目标是构建一个长期可演进的小说智能打分系统，因此需要优先确定目录边界，而不是过早绑定具体技术栈。

## 决策

采用如下正式顶层结构：

- `apps/`
- `packages/`
- `prompts/`
- `evals/`
- `docs/`
- `output/`
- `scripts/`

该结构以职责边界为核心，不以具体前端或后端实现技术命名。

## 结果

- 有利于后续技术选型变更
- 有利于分离应用入口、领域逻辑、Prompt、评测与研究产物
- 为长期演进提供稳定结构基础
