# 面向网络小说双输入 Rubric 的设计吸收矩阵

> 历史说明：本文形成于早期设计收口阶段，仍保留“旧四维骨架层 / 对外四分字段”的方案假设。当前代码实现已经收敛为 `overall + axes + optional typeAssessment` 正式结果，并新增解耦的 `type_classification / type_lens_evaluation` 主线；`aggregation` 不再输出旧四维骨架，对外 API 也不再返回四分字段。阅读本文时，请以 `docs/architecture/scoring-pipeline.md`、`docs/architecture/layered-rubric-evaluation-architecture.md` 与 `docs/contracts/json-contracts.md` 作为当前实现真源。

## 目标

本文档用于汇总当前正式评分主线中必须固化的设计要求，并为 Prompt 资产、Schema、Evals 与运行时编排提供统一设计输入。

本文档不承担历史对照职责，也不保留旧正式主线路径，只服务于当前面向网络小说投稿评估的双输入正式主线收口。

## 背景

当前正式评分主线采用：

- 输入预检查
- `LLM rubric` 分点评价
- 轻量一致性整理
- 新模型聚合输出
- 正式结果投影

在此前主线基础上，当前设计进一步冻结为：

- 正式输入采用 `chapters + outline` 联合模型
- 正式评分只保留 `pointwise`
- 新 `8` 轴作为主评价层
- 旧四维保留为中间骨架层
- `fatalRisk` 作为跨层约束维度
- 对外继续保留稳定四分字段

因此，设计收口的关键不在于继续堆叠宽泛描述，而在于把关键要求落到正确层次，形成可治理、可评测、可直接支持开发的正式资产。

## 分类规则

### 一类：必须固化

定义：

- 若缺失会直接破坏产品辨识度
- 若缺失会破坏对外结果消费预期
- 若缺失会导致评分哲学失真

落位方式：

- 应固化到正式架构文档、Rubric 规则、输出约束或正式 Schema 中

### 二类：必须结构化

定义：

- 要求本身有效，但不能停留在抽象自然语言
- 需要拆成阶段规则、词表、错误语义、阶段契约或聚合约束

落位方式：

- 应改写为可执行、可验证、可版本化的正式资产

### 三类：禁止回流

定义：

- 会破坏单主线边界
- 会重新引入旧路线中的双路径、仲裁或人工流程
- 会造成双真源或职责混乱

落位方式：

- 不进入当前正式主线
- 在设计、实现与评测中都应显式避免

## 设计吸收矩阵

| 设计要求 | 分类 | 正式落点 | 说明 |
| --- | --- | --- | --- |
| 正式输入以 `chapters + outline` 为中心 | 必须固化 | 架构文档 + API 契约 + 前端最小提交对象 | 决定系统面向投稿包而不是单文本 |
| `chapters only` 与 `outline only` 允许降级评估 | 必须固化 | 输入预检查契约 + 错误语义 | 必须显式表达输入充分性 |
| 正式评分只保留 `pointwise` | 必须固化 | ADR + 架构文档 + Prompt 治理 | 正式主线不再引入 `pairwise` |
| 新 `8` 轴为主评价层 | 必须固化 | 架构文档 + 阶段契约 | 冻结主评价层口径 |
| 旧四维保留为中间骨架层 | 必须固化 | 架构文档 + JSON 契约说明 | 保留稳定映射骨架 |
| 对外四分字段保持稳定 | 必须固化 | `packages/schemas/output/` + JSON 契约 | 属于对外消费契约 |
| `platformFit` 独立成轴 | 必须固化 | 新 `8` 轴定义 + 聚合规则 | 不再降级为市场子项 |
| `fatalRisk` 跨层约束保留 | 必须固化 | 阶段契约 + 聚合规则 | 不作为普通加权项 |
| 正文与大纲要进行联合判断 | 必须固化 | 输入预检查 + 一致性整理契约 | 不能拆成松散独立任务 |
| 跨输入冲突必须结构化表达 | 必须结构化 | `ConsistencyCheckResult` + 错误语义 | 用于承诺冲突与兑现不足识别 |
| 证据要区分 `chapters` / `outline` / `cross_input` | 必须结构化 | `RubricEvaluationEvidenceRef` | 便于评测与聚合追踪 |
| 标签机制保持轻量、封闭、少量 | 必须结构化 | Prompt registry + 词表治理 | 允许条件化，不允许流程分叉 |
| 女频不再拆出复杂子流程 | 必须固化 | 轻量标签治理说明 | 避免过度结构化 |
| `signingProbability` 不得脱离基础分独立飙升 | 必须固化 | 聚合约束规则 | 保证顶层字段逻辑稳定 |
| `strengths` / `weaknesses` 保持短项输出 | 必须固化 | 输出 Schema + 最终投影约束 | 维持消费友好性 |
| `editorVerdict` 为一句话标签 | 必须固化 | 输出 Schema + 聚合规则 | 属于产品辨识度特征 |
| `marketFit` 为简短市场定位 | 必须固化 | 输出 Schema + 聚合规则 | 避免空泛总结 |
| 平台推荐必须排序，首位为最优平台 | 必须固化 | 输出 Schema + 聚合规则 | 与消费逻辑直接相关 |
| AI 味、术语污染、套路拼贴需要重罚 | 必须结构化 | `fatalRisk` 词典 + 负样本集 | 需要可评测的正式风险目录 |
| 假兑现、空承诺、大纲补偿正文失败需要识别 | 必须结构化 | `fatalRisk.fakePayoff` + 一致性整理 | 属于双输入场景核心风险 |
| 不可评、结构失败、模型失败必须区分 | 必须结构化 | 错误语义 + 任务生命周期 | 不能混成一个失败状态 |
| 前端直接传正式 Prompt | 禁止回流 | Prompt 治理边界 | 与项目约束冲突 |
| `pairwise` 成为正式评分路径 | 禁止回流 | 单主线治理 | 与当前需求冲突 |
| 为不同题材拆成多套主线流程 | 禁止回流 | 轻量标签治理 | 会破坏可治理性 |
| 在多个位置维护独立评分真源 | 禁止回流 | 单一 Rubric 真源治理 | 会造成双真源 |
| 把产品化措辞前置到所有阶段 | 禁止回流 | 仅保留在聚合与最终投影层 | 会降低中间阶段可治理性 |

## 新 `8` 轴定义吸收

| 轴 ID | 吸收的旧价值 | 当前正式定义 |
| --- | --- | --- |
| `hookRetention` | 旧市场吸引力中的开场抓力与留存动机 | 判断读者是否愿意继续读下去 |
| `serialMomentum` | 旧追更潜力与冲突延续判断 | 判断连载惯性和后续拉力是否成立 |
| `characterDrive` | 旧人物推动力与情绪牵引 | 判断角色欲望、行动性与关系张力 |
| `narrativeControl` | 旧叙事执行与语言控制 | 判断正文执行力与表达稳定性 |
| `pacingPayoff` | 旧节奏控制与兑现关系 | 判断铺垫、推进与回报是否匹配 |
| `settingDifferentiation` | 旧新鲜度与设定可用性 | 判断设定是否形成有效差异化 |
| `platformFit` | 旧市场判断中的平台适配部分 | 判断内容与平台读者预期的匹配程度 |
| `commercialPotential` | 旧签约潜力与商业判断 | 判断作品是否具备连载转化潜力 |

## 旧四维骨架吸收

| 骨架维度 | 当前职责 |
| --- | --- |
| `marketAttraction` | 汇总抓力、连载动能、平台适配与商业潜力 |
| `narrativeExecution` | 汇总叙事执行、节奏控制与信息组织 |
| `characterMomentum` | 汇总人物驱动、关系张力与情绪牵引 |
| `noveltyUtility` | 汇总设定差异化、新鲜度与卖点可用性 |

说明：

- 旧四维不再是最上层评分入口
- 旧四维继续承担稳定映射骨架职责

## 禁止回流边界

以下内容不应进入当前正式主线：

- 把正文和大纲拆成两个独立正式评分任务
- 让 `pairwise` 回流为正式评分路径
- 为不同题材维护不同主线流程
- 在多个位置分别维护独立评分规则真源
- 在没有阶段契约的情况下绕过结构化中间产物
- 重新引入升级分流、外部仲裁或保守模式
