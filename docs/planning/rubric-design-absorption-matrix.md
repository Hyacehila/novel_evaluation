# 分层 Rubric 设计吸收矩阵

## 目标

本文档用于汇总当前正式评分主线中必须固化的设计要求，并为 Prompt 资产、Schema、Evals 与运行时编排提供统一设计输入。

本文档不承担历史对照职责，也不保留旧主线路径，只服务于当前正式主线的设计收口。

## 背景

当前正式评分主线采用“分层 `rubric` + 小模型优先 + 升级仲裁”的机制。

其核心不是让单个黑盒评分节点完成全部判断，而是把评价能力拆分为以下层次：

- 输入预检查
- 证据抽取
- 原子评分
- 一致性校验
- 聚合与校准
- 升级仲裁

因此，设计收口的关键不在于堆叠更长的规则文本，而在于把关键要求放到正确的层次上，并形成可治理、可评测、可演进的正式资产。

## 分类规则

### 一类：必须固化

定义：

- 若缺失会直接破坏产品辨识度
- 若缺失会破坏对外结果消费预期
- 若缺失会导致评分哲学失真

落位方式：

- 应被固化到系统级 Prompt、Rubric 规则、输出约束或正式 Schema 中

### 二类：必须结构化

定义：

- 要求本身有效，但不能只停留在抽象自然语言描述
- 需要拆成阶段规则、词典、阶段契约或校准逻辑

落位方式：

- 应被改写为可执行、可验证、可版本化的正式资产

### 三类：禁止回退

定义：

- 会破坏分层边界
- 会重新引入黑盒单点评价
- 会造成双真源或职责混乱

落位方式：

- 不进入当前正式主线
- 在设计、实现和评测中都应显式避免

## 设计吸收矩阵

| 设计要求 | 分类 | 正式落点 | 说明 |
| --- | --- | --- | --- |
| 总编 / 主编视角 | 必须固化 | `prompts/scoring/system/` | 决定系统人格与判断视角 |
| 锋利、直接、短促的编辑口吻 | 必须固化 | `prompts/scoring/system/` + 最终投影层约束 | 只在最终结果层显性表达，不污染中间阶段 |
| 不软化结论 | 必须固化 | `prompts/scoring/system/` | 属于结果风格约束 |
| 重点判断“能不能签、能不能跑平台、能不能留读者” | 必须固化 | 一级维度与聚合原则 | 属于评分哲学核心 |
| 四个顶层分数保持稳定 | 必须固化 | `packages/schemas/output/` + 聚合层 | 属于外部结果结构契约 |
| `signingProbability` 依赖三项基础分且不能飙高 | 必须固化 | 聚合校准规则 | 属于顶层分数逻辑约束 |
| 市场吸引力强调 hook、追更、平台适配 | 必须固化 | `marketAttraction` | 构成商业判断主骨架 |
| 写作质量强调执行力而非文学评论 | 必须固化 | `narrativeExecution` + 系统级 Prompt | 明确可读性与执行优先 |
| 创新强调“可用的新”而不是猎奇乱炖 | 必须固化 | `noveltyUtility` + 校准规则 | 无效怪异不应被视为创新 |
| 非小说文本或不可评文本识别 | 必须固化 | 输入预检查层 | 不应由最终评分兜底 |
| `strengths` / `weaknesses` 保持短项输出 | 必须固化 | 最终投影层结构约束 | 需明确条数、长度和粒度 |
| `editorVerdict` 为一句话标签 | 必须固化 | 最终投影层结构约束 | 属于高辨识度产品特征 |
| `marketFit` 为简短市场定位 | 必须固化 | 最终投影层结构约束 | 避免泛泛总结 |
| 平台推荐必须排序，首位即最优平台 | 必须固化 | `platforms` 输出约束 | 与结果消费逻辑直接相关 |
| `platforms` 数量和百分比约束 | 必须固化 | 输出 Schema + 校准规则 | 数量、排序、百分比应可校验 |
| 四段 `detailedAnalysis.*` 各自负责固定主题 | 必须固化 | 最终投影层结构约束 | 必须保留职责分工 |
| system 负责规则宪法，user 只注入文本 | 必须固化 | Prompt 治理规范 | 属于 message discipline |
| 严格 JSON only | 必须固化 | 所有阶段契约 + 最终输出 Schema | 属于基础约束 |
| Prompt 版本记录与样本绑定 | 必须固化 | Prompt registry + evals | 属于治理能力要求 |
| AI 味 / 教程腔 / 说明书腔重罚 | 必须结构化 | `fatalRisk` 风险词典 | 需要正式风险目录 |
| 术语轰炸 / 命名污染 / 系统面板污染重罚 | 必须结构化 | `fatalRisk` 风险词典 | 应在抽取层与校准层可见 |
| 低级套路 / 老梗拼贴重罚 | 必须结构化 | `fatalRisk.staleFormula` | 需要细化触发词典 |
| 长段自然语言惩罚描述 | 必须结构化 | 风险标签词典 + 校准规则 | 便于评测与版本治理 |
| 不可评 / 结构失败 / 模型失败混合处理 | 必须结构化 | 结构化错误语义 | 应进入阶段契约与错误类型 |
| 将证据、评分、校准、措辞、失败语义混在一个黑盒节点 | 禁止回退 | 分阶段编排 | 会破坏主线分层边界 |
| 让一个评分节点承担全部工作 | 禁止回退 | 多阶段编排 | 属于单点黑盒模式 |
| 允许多个位置分别维护独立评分真源 | 禁止回退 | 单一 rubric 真源治理 | 会造成双真源 |
| 把产品化措辞前置到所有阶段 | 禁止回退 | 仅保留在最终投影层 | 会降低中间阶段可治理性 |

## 当前尚待补成正式资产的内容

### 1. 产品语气约束

需要补成正式资产：

- 严厉主编口吻
- 不软化结论
- 锋利但不失真
- 判词短促、可消费

建议落位：

- `prompts/scoring/system/`
- 相关 Prompt 版本文档

### 2. 输出微规则

需要补成正式资产：

- `strengths` / `weaknesses` 的条数、长度、直接性
- `editorVerdict` 的一句话约束
- `marketFit` 的句长与定位约束
- `platforms` 的数量、排序、百分比约束
- `detailedAnalysis.*` 的职责边界

建议落位：

- `packages/schemas/output/`
- 最终投影层规则文档
- 校准 Prompt

### 3. 风险词典

需要补成正式资产：

- 低级钩子
- 打脸空转冲突
- 主角纸片化
- AI 味 / 说明书腔
- 术语污染 / 命名污染 / 系统面板污染
- 旧套路复写

建议落位：

- `prompts/scoring/rubric/`
- `prompts/extraction/`
- `prompts/calibration/`
- `evals/` 负样本集

### 4. Message Discipline

需要补成正式资产：

- system 负责规则宪法
- user 不得重写评分规则
- 输出不得附带 JSON 外自然语言
- 最终投影层才负责产品化措辞

建议落位：

- Prompt 治理规范
- Prompt registry

## 禁止回退边界

以下内容不应进入当前正式主线：

- 让单个黑盒评分节点承担全部职责
- 将证据、评分、校准、措辞混在同一节点处理
- 将产品化措辞前置到所有阶段
- 在多个位置分别维护独立评分规则真源
- 在没有阶段契约的情况下绕过结构化中间产物

## 输出要求

当前设计收口至少应保证：

- 一份正式的设计吸收矩阵文档
- 一份必须固化要求清单
- 一份必须结构化要求清单
- 一份禁止回退边界清单
- Prompt / Schema / Evals / 运行时编排的明确落点说明

- `Phase 1` 基于本文档冻结正式主线的设计要求
- `Phase 2` 基于本文档建设 Prompt 资产
- `Phase 3` 基于本文档冻结 Schema 与阶段契约
- `Phase 4` 基于本文档设计 Evals 与回归口径
