# 前端页面规格说明

## 文档定位

本文定义首期前端核心页面的页面级规格，供后续前端开发、接口联调、交互细化与测试设计使用。

本文收敛原“页面规格说明”“列表与详情展示规则”“交互与空态规范”“历史记录与检索策略”“可视化与评分呈现规范”的页面规划层内容，聚焦页面职责、页面模块、展示边界、异常场景与首期检索规则。

本文不展开组件实现、视觉稿或框架实现细节。

## 适用范围

本文覆盖首期核心页面：

- `工作台首页`
- `新建评测任务页`
- `任务详情 / 状态页`
- `结果详情页`
- `历史记录页`

本文不覆盖：

- `结果对比页` 的详细规格
- 组件级设计
- 具体视觉布局
- 具体实现方案

## 全局页面规格原则

### 1. 页面级优先

首期规格以页面为基本单元，优先明确每页的职责边界与输入输出，不将当前阶段过度细化到组件层。

### 2. 契约优先

结果相关页面只展示后端已经校验过的结构化结果。

### 3. 任务流优先

页面之间围绕“新建任务 -> 查看状态 -> 查看结果 -> 历史回访”的主流程组织。

### 4. 任务对象与结果对象分离

- `任务详情 / 状态页` 聚焦任务生命周期
- `结果详情页` 聚焦结构化结果阅读

### 5. 摘要层与详情层分离

- 首页与历史记录页只展示摘要，不替代完整详情页
- `任务详情 / 状态页` 展示任务过程，不展示正式结果正文
- `结果详情页` 展示完整正式结果，不承担任务过程解释

### 6. 首期基础元信息展示

首期页面中显式展示的基础元信息优先包括：

- `任务 ID`
- `创建时间`
- `结果时间`

更完整的 Prompt、Schema、Provider、结果版本元信息留待后续阶段扩展。

## 页面一：工作台首页

### 页面目标

作为系统统一入口，平衡展示任务入口与结果入口，帮助用户快速开始新任务或回到最近的任务与结果。

### 页面输入

- 最近任务摘要列表
- 处理中任务摘要列表
- 最近结果摘要列表

### 页面输出

- 进入 `新建评测任务页` 的入口
- 进入 `任务详情 / 状态页` 的入口
- 进入 `结果详情页` 的入口
- 进入 `历史记录页` 的入口

### 建议页面模块

- 快速新建评测区
- 最近任务摘要区
- 处理中任务摘要区
- 最近结果摘要区
- 历史记录快捷入口

### 展示边界

可展示：

- 最近任务摘要
- 处理中任务状态
- 最近结果摘要
- 基础元信息

不展示：

- 完整详细分析
- 完整平台推荐详情
- 完整优弱项列表
- 任务失败的深度解释

### 页面状态与异常场景

- 首页已加载
- 首页无最近任务
- 首页无最近结果
- 存在处理中任务
- 不存在处理中任务
- 最近任务读取失败
- 最近结果读取失败
- 首页摘要信息为空

说明：

- 首页无摘要信息时，应视为正常空态，不应直接视为系统错误。
- 最近结果摘要仅作为快捷入口，不替代结果详情页。

### 页面跳转关系

- 跳转至 `新建评测任务页`
- 跳转至 `任务详情 / 状态页`
- 跳转至 `结果详情页`
- 跳转至 `历史记录页`

## 页面二：新建评测任务页

### 页面目标

承载用户输入评测内容并创建任务，是系统正式任务流的起点。

### 页面输入

- 标题
- 章节正文输入
- 大纲输入
- 文件上传内容
- 来源类型

### 页面输出

- 新建任务请求
- 新建成功后的任务 ID

### 建议页面模块

- 标题输入区
- 章节输入区
- 大纲输入区
- 文件上传区
- 输入摘要与校验提示区
- 提交操作区

### 展示边界

可展示：

- 联合投稿包输入内容
- 文件信息
- `inputComposition` 与 `evaluationMode` 的派生提示
- `sourceType`
- 校验提示
- 提交状态

不展示：

- 任务执行状态
- 正式结果正文
- Prompt 正文
- Prompt 版本选择
- Provider 内部策略

说明：

- 首期基础业务参数仅包括 `sourceType` 等非 Prompt 型参数。
- `inputComposition` 和 `evaluationMode` 属于输入后的派生提示，不是独立旧输入枚举。

### 页面状态与异常场景

- 输入中
- 校验失败
- 提交中
- 提交成功
- 提交失败
- 标题为空
- `chapters` 与 `outline` 同时缺失
- 文件上传失败
- 提交任务失败

### 页面跳转关系

- 提交成功后跳转至 `任务详情 / 状态页`
- 取消或返回可跳转至 `工作台首页`

## 页面三：任务详情 / 状态页

### 页面目标

展示单任务的执行状态、过程信息与基础元信息，帮助用户理解任务当前所处阶段，并在条件满足时进入结果页。

### 页面输入

- 任务标识
- 输入摘要
- 输入组成
- 评估模式
- 任务状态
- 错误信息
- 基础元信息

### 页面输出

- 结果详情页入口
- 历史记录页返回入口
- 工作台首页返回入口

### 建议页面模块

- 任务头部信息区
- 状态展示区
- 输入摘要区
- 任务元信息区
- 错误信息区
- 结果入口区

### 展示边界

可展示：

- 任务状态
- 输入摘要
- `inputComposition`
- `evaluationMode`
- 基础元信息
- 错误信息
- 结果入口

不展示：

- 四项评分完整内容
- 详细分析正文
- 优弱项正文
- 平台推荐完整列表

### 页面状态与异常场景

- `queued`
- `processing`
- `completed`
- `failed`
- 任务不存在
- 任务状态查询失败
- 任务失败且无结果
- 任务完成但结果暂不可用
- 任务完成但结果被阻断

### 页面跳转关系

- 任务完成且结果可用时，可进入 `结果详情页`
- 可返回 `历史记录页`
- 可返回 `工作台首页`

说明：

- 当前策略是任务完成后默认留在本页，由用户手动进入 `结果详情页`。

## 页面四：结果详情页

### 页面目标

承载结构化评测结果的完整阅读与消费，是系统结果对象的主视图。

### 页面输入

- 顶层评分
- 平台推荐
- 编辑结论
- 市场判断
- 详细分析
- 优势与弱点
- 可视化数据
- 基础元信息

### 页面输出

- 结果阅读
- 回到任务详情页入口
- 回到历史记录页入口
- 未来对比入口预留

### 建议页面模块

- 顶层评分摘要区
- 平台推荐区
- 编辑结论区
- 市场判断区
- 详细分析区
- 优势与弱点区
- 可视化区
- 结果元信息区

### 展示边界

必须围绕正式契约展示：

- 四个顶层评分
- 平台推荐
- 编辑结论
- 市场判断
- 详细分析
- 优势与弱点

可辅助展示：

- 可视化内容

规则：

- 若存在可视化所需数据，则可展示
- 若不存在可视化数据，不强行构造图表
- 可视化是辅助阅读层，不替代正式结果内容
- 数值展示应与正式结果字段保持一致
- 不得对后端未提供的数值做推断或补全

### 页面状态与异常场景

- `available`
- `not_available`
- `blocked`
- `fetch_failed`（前端本地派生状态）
- 结果不存在
- 结果读取失败
- 结果尚未生成
- 结果不允许显示

### 页面跳转关系

- 返回 `任务详情 / 状态页`
- 返回 `历史记录页`
- 返回 `工作台首页`
- 未来进入 `结果对比页`

### 异常展示限制

- 阻断结果不得进入结果详情正文
- 结果不可用不得伪装成“部分结果页”
- 不得将错误态包装为正常结果阅读页

## 页面五：历史记录页

### 页面目标

以任务为主列表对象沉淀历史评测记录，支持用户回访过去任务与对应结果。

### 页面输入

- 历史任务列表
- 任务状态
- 输入组成
- 创建时间
- 结果可用性
- 首期筛选条件

### 页面输出

- 进入 `任务详情 / 状态页` 入口
- 进入 `结果详情页` 入口
- 条件筛选结果

### 建议页面模块

- 筛选与检索区
- 历史任务列表区
- 状态标识区
- 进入任务详情页入口
- 进入结果详情页入口

### 展示边界

可展示：

- 任务 ID
- 标题或输入摘要
- 输入组成
- 状态
- 创建时间
- 结果可用性

不展示：

- 完整结果正文
- 详细分析
- 可视化主内容
- 结果对比能力

### 页面状态与异常场景

- 列表已加载
- 列表为空
- 筛选中
- 筛选后为空
- 列表加载失败
- 历史记录读取失败
- 筛选条件无结果
- 任务存在但结果不可用

### 首期检索规则

首期优先支持：

- `任务 ID` 搜索
- `标题/输入摘要` 搜索
- 任务状态筛选

首期不要求：

- 全文检索
- 时间范围筛选
- 输入组成筛选
- Provider、版本、结果维度筛选

### 列表规则

- 主列表对象为 `HistoryTaskItemView`
- 建议按创建时间倒序展示最近任务
- 若结果可用，应提供进入结果页入口
- 若结果不可用，应优先保持任务视角，而不是强行展示结果摘要

### 页面跳转关系

- 进入 `任务详情 / 状态页`
- 在结果可用时进入 `结果详情页`
- 返回 `工作台首页`
