# 快速开始

本文档面向第一次运行项目的用户，目标是在几分钟内完成本地启动和一次评测提交。

## 运行前准备

当前官方验证环境为 `Windows + PowerShell`。请先安装：

- `Python 3.13`
- [`uv`](https://docs.astral.sh/uv/)
- `Node.js 20+`
- `pnpm`

## 1. 安装依赖

在仓库根目录执行：

```powershell
.\scripts\setup.ps1
```

该脚本会：

- 安装 `apps/api` 依赖
- 安装 `apps/worker` 依赖
- 安装 `apps/web` 依赖
- 若仓库根目录没有 `.env`，自动从 `.env.example` 复制一份

## 2. 启动 API

新开一个 PowerShell 窗口，在仓库根目录执行：

```powershell
.\scripts\run-api.ps1
```

默认地址：`http://127.0.0.1:8000`

## 3. 启动 Web

再开一个 PowerShell 窗口，在仓库根目录执行：

```powershell
.\scripts\run-web.ps1
```

默认地址：`http://127.0.0.1:3000`

## 4. 提交第一份评测

打开 `http://127.0.0.1:3000/tasks/new`。

如果 API 启动时已经配置了 `NOVEL_EVAL_DEEPSEEK_API_KEY`，可以直接填入标题、正文和大纲创建任务。

如果没有配置 API Key，API 仍可启动，但此时只能查看已有数据，不能直接创建新分析任务。你可以先在页面中录入一次性 runtime key，再提交任务；该 key 只保存在当前 API 进程内，重启或热重载后失效。

示例正文：

```text
第一章里，青岚宗护山大阵突然熄灭，外门弟子顾辞在山门值守时亲眼看见长老重伤归来。执法堂宣布七天后宗门大比照常举行，若无人夺得秘境名额，宗门将被邻宗吞并。顾辞为了保住师父留下的药园，只能冒险借用残缺功法突破境界，同时调查阵法失效的真相。短短一夜里，他先后失去住处、同伴和退路，被迫主动卷入更大的权力斗争。
```

示例大纲：

```text
后续主线围绕宗门大比、秘境夺宝和宗门重建展开。顾辞会先在大比中证明自己，再进入秘境寻找修复大阵的核心灵物，同时逐步揭开长老叛变与外敌联手的内幕。中期通过师徒关系、同门竞争和资源争夺持续升级冲突，后期把宗门存亡、个人成长和秘宝兑现收束到同一条升级主线上。
```

## 5. 你应该看到什么

- 新建任务后会跳到任务详情页
- 任务状态会从 `queued / processing` 进入 `completed` 或 `failed`
- 任务执行中，任务页会先显示“类型识别”区域；类型判出后会显示类型 badge、识别置信度和是否触发兜底
- 结果可用时，结果页会展示：
  - 总体评分、总体结论、平台候选和市场判断
  - 独立的类型评价模块
  - 当前识别类型对应的 `4` 个 lens 卡片
  - `8` 个 rubric 评价轴
- 结果被阻断或结果不可用时，结果页只展示语义化提示，不展示伪结果正文

若 API 已有可用 provider key，这次结果会走真实 Provider。若 API 启动时无 key，则页面会提示先录入一次性 runtime key；未录入前只能查看已有任务、结果与历史，不能新建分析任务。

如果你之前已经有旧版本数据库，历史结果可能因为仍是旧 schema 而显示为 `not_available`。这是当前实现的兼容策略，重新提交一次任务即可生成新结构结果。

## 自定义配置

如果你想改端口、数据库路径或日志目录：

```powershell
Copy-Item .env.example .env
```

然后修改 `.env` 中对应的变量，再重新启动脚本。
