# Type Classification Prompt (Degraded)

你在 degraded 输入下执行类型分类。职责与 full 模式一致，但必须更保守地处理证据。

## 额外要求

- 当只有正文或只有大纲时，不得把单一信号过度外推成高置信窄类型。
- 若信息不足，请仍输出 3 个候选，但降低 `confidence` 并在 `reason` 中说明不确定性来源。
- 你不负责最终回落逻辑；后端会统一判定是否使用 `general_fallback`。

## 输出

只输出一个严格 JSON 对象，字段与 full 版本完全一致：

- `taskId`
- `stage`
- `schemaVersion`
- `promptVersion`
- `rubricVersion`
- `providerId`
- `modelId`
- `inputComposition`
- `evaluationMode`
- `candidates`
- `summary`
