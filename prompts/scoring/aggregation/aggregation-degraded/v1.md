# Aggregation Prompt (Degraded)

你负责 `aggregation` 阶段的 `degraded` 模式。此时输入可能只有正文或只有大纲，或者 `screening` 已明确材料不足。你的任务是给出保守、结构化、可落地的聚合 JSON。

## 输出

只返回一个 JSON object。字段必须严格限制为：

- `taskId`
- `stage`
- `schemaVersion`
- `promptVersion`
- `rubricVersion`
- `providerId`
- `modelId`
- `overallVerdictDraft`
- `verdictSubQuote`
- `overallSummaryDraft`
- `platformCandidates`
- `marketFitDraft`
- `strengthCandidates`
- `weaknessCandidates`
- `riskTags`
- `overallConfidence`

## 降级模式原则

- 明确承认输入受限，不要把局部证据写成全量判断
- 若关键依据缺失，应在 `overallSummaryDraft` 和 `marketFitDraft` 中直接说明
- 若 `screening` 或 `rubric` 已显示 `insufficientMaterial`，默认保留到 `riskTags`
- `overallConfidence` 应明显保守
- `overallVerdictDraft` 可以建议“补充正文后复核”或“补充大纲后复核”，但不要给出过强承诺

## 字段细则

- `overallVerdictDraft`：1 句保守主判断
- `verdictSubQuote`：可为 `null`；若填写，用于补充材料限制或复核条件
- `overallSummaryDraft`：2 到 4 句，说明能判断什么、不能判断什么
- `platformCandidates`：`[]` 或 1 到 2 个保守候选；若非空，权重总和必须为 `100`
- `marketFitDraft`：说明当前输入能支持到什么程度的圈层/平台判断
- `strengthCandidates` / `weaknessCandidates`：各自 `[]` 或 1 到 3 条，全部来自现有输入
- `riskTags`：只能使用正式枚举值
- `overallConfidence`：`0` 到 `1`

## 禁止项

- 不要输出旧字段、旧别名或额外字段
- 不要输出历史四分类顶层分数
- 不要输出旧骨架字段
