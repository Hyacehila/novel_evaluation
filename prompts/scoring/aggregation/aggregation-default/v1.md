# Aggregation Prompt (Full)

你负责 `aggregation` 阶段。目标不是重新打分，而是把已有阶段结果收束成一个可直接进入 `final_projection` 的聚合 JSON。

## 输入

用户消息会提供：

- `screening`
- `typeClassification`
- `rubric`
- `typeLens`
- `consistency`
- 原始正文与大纲

请只基于这些输入作答，不要补写不存在的事实。

## 输出

只返回一个 JSON object，不要输出代码块、解释或额外文本。字段必须严格限制为：

- `taskId`
- `stage`
- `schemaVersion`
- `promptVersion`
- `rubricVersion`
- `providerId`
- `modelId`
- `overallVerdictDraft`
- `verdictSubQuote`
- `overallSummaryDraft`
- `platformCandidates`
- `marketFitDraft`
- `strengthCandidates`
- `weaknessCandidates`
- `riskTags`
- `overallConfidence`

## 字段要求

### `overallVerdictDraft`

- 1 句中文主判断
- 必须可读、明确、与 8 轴和类型 lens 一致
- 允许保守，但不能空泛

### `verdictSubQuote`

- 可为 `null`
- 若填写，只写 1 句补充条件或市场补充判断
- 不要重复主判断

### `overallSummaryDraft`

- 2 到 4 句
- 概括当前材料的整体完成度、主要亮点、主要短板和后续观察点

### `platformCandidates`

- `[]` 或 1 到 3 个候选
- 每个候选必须是 `{ "name": string, "weight": int, "pitchQuote": string }`
- `weight` 为 `0-100` 整数
- 若非空，所有 `weight` 之和必须为 `100`

### `marketFitDraft`

- 1 到 3 句
- 解释圈层、平台或市场匹配度

### `strengthCandidates` / `weaknessCandidates`

- 各自 `[]` 或 1 到 3 条
- 每条是短语或短句
- 必须和 `rubric` / `typeLens` 中的证据一致

### `riskTags`

只能从以下值中选择，且不重复：

- `aiManualTone`
- `staleFormula`
- `conceptSpam`
- `fakePayoff`
- `nonNarrativeSubmission`
- `insufficientMaterial`

### `overallConfidence`

- `0` 到 `1`
- 反映整体判断稳定度
- 不能高于你从输入中能支撑的把握

## 强约束

- 不要输出任何旧字段或兼容别名
- 不要输出历史四分类顶层分数
- 不要输出旧骨架相关字段
- 不要输出额外字段
- 不要把阻断或低置信输入包装成乐观结论
