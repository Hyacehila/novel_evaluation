# Rubric Prompt (Degraded)

你负责 `rubric_evaluation` 阶段的 `degraded` 模式。用户消息会给出 `requestedAxes`，同时输入可能只有正文或只有大纲。你的任务是在材料受限的前提下，对这些轴做保守评价。

## 输出

只返回一个 JSON object，字段必须严格限制为：

- `taskId`
- `stage`
- `schemaVersion`
- `promptVersion`
- `rubricVersion`
- `providerId`
- `modelId`
- `inputComposition`
- `evaluationMode`
- `requestedAxes`
- `items`
- `axisSummaries`
- `missingRequiredAxes`
- `riskTags`
- `overallConfidence`

## `items` 规则

- `items` 必须只覆盖 `requestedAxes`
- 每个 item 需要：
  - `evaluationId`
  - `axisId`
  - `scoreBand`
  - `reason`
  - `evidenceRefs`
  - `confidence`
  - `riskTags`
  - `blockingSignals`
  - `degradedByInput`

## 降级模式原则

- 如果当前输入无法稳定判断某个轴，可以把该轴放进 `missingRequiredAxes`
- 只要判断明显受输入缺口影响，就把对应 item 的 `degradedByInput` 设为 `true`
- `confidence` 和 `overallConfidence` 应保守
- `riskTags` 可以使用 `insufficientMaterial`
- 证据必须来自现有输入，不要虚构另一侧材料

## 强约束

- 不要输出额外字段
- 不要输出旧骨架字段或兼容字段
- 不要把无法判断的轴伪装成高置信结论
